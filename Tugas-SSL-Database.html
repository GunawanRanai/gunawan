Penerapan Koneksi Ter-Enkripsi pada MySQL

  • Gunawan
  • Sep 23, 2022

Pendahuluan

Pada MySQL bisa menerapkan SSL (Secure Socket Layer) dan juga TLS (Transport Layer Security). SSL sendiri merupakan protokol untuk memastikan keamanan saat komunikasi antara client dan server. Protokol ini menggunakan sertifikat untuk memverifikasi pihak yang berkomunikasi. TLS sendiri didefinisikan dalam suatu request for comment pada RFC2246. Protokol yang menggunakan TLS untuk menjaga keamanan koneksi antara lain, HTTP, IMAP, POP3 dan SMTP.

Pada MySQL sendiri jika menggunakan komunikasi antara client dan server yang berbeda mesin, maka akan otomatis menggunakan protokol SSL yang diaktifkan melalui jenis TLS yang dipakai. Pada MySQL Sendiri sudah bisa menggunakan TLSv.1.3 yang secara otomatis menggunakan SSL dengan jenis enkripsi TLS_AES_256_GCM_SHA384.

Implementasi Koneksi Ter-Enkripsi pada Database Secara Remote

Adapun proses untuk menggunakan koneksi aman pada MySQL sendiri sebagai berikut.

1. Pada MySQL Sendiri sudah memiliki sertifikat untuk menjalankan koneksi ter-enkripsi. Sehingga langsung saja menambahkan atau mengaktifkan koneksi ter-enkripsi pada MySQL dengan menuju lokasi file dengan perintah berikut.

sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf

file ini terdapat pada path /var/lib/mysql

Jika file belum ada maka bisa menggunakan perintah berikut.

sudo mysql_ssl_rsa_setup -uid=mysql

2. Selanjutnya kita bisa memverifikasi bahwa SSL pada MySQL sudah terinstall dengan menggunakan perintah berikut.

show variables like '%ssl%';

3. Selanjutnya bisa kita cek koneksi yang terjadi pada MySQL dengan perintah \s.

Perbandingan Antara Penggunaan HTTP dan HTTPS Menggunakan WireShark dan tcpdump

Adapun untuk mengukur perbandingan keamanan saat menggunakan HTTP dan HTTPS akan dilakukan dengan memanfaatkan tools tcpdump dan software WireShark untuk melihat proses komunikasi antara client dan server MySQL.Langkah – langkah yang akan dilakukan sebagai berikut.

sudo tcpdump -i [interface] -s 65539 -w [nama_file].

pada HTTP atau tanpa SSL sebagai berikut. sudo tcpdump -i enp0s8 -s 65539 -w analisis_http.pcap

Pada terminal lainnya kita akan login ke MySQL dengan menggunakan option --ssl-mode=DISABLED, dan lakukan query MySQL seperti biasa.

Hal ini bertujuan agar koneksi yang digunakan tidak menggunakan SSL atau TLS sehingga bisa kita lakukan perbandingan antara koneksi HTTP dan HTTPS.

Dengan hasil penyadapan komunikasi seperti gambar dibawah ini.

Pada HTTPS atau penggunaan SSL menggunakan perintah berikut.
sudo tcpdump -i enp0s8 -s 65539 -w analisis_https.pcap


Maka akan didapatkan data komunikasi sebagai berikut.

Dimana saat menggunakan HTTP protokol MySQL sangat jelas terlihat, namun saat menggunakan HTTPS atau SSL terlihat komunikasi pada MySQL terlindungi oleh TLS. Dengan perbedaan yang dimiliki sebagai berikut

Perbandingan Performa Antara Penggunaan HTTP dan HTTPS

Pengukuran perbandingan performa antara penggunaan MySQL secara HTTP dan HTTPS bisa dilakukan dengan memanfaatkan tools mysqlslap. Mysqlslap merupakan tools yang dirancang untuk menirukan beban klien pada server MySQL dan melaporkan waktu setiap tahapan yang dilewati. Berikut tahapan dan hasil yang dilaksanakan.

Perintah yang digunakan untuk mengukur performasi pada HTTP sebagai berikut.

sudo mysqlslap --user=web1 --password --host=192.168.2.4 --concurrency=100 --iterations=100 --number-int-cols=5 --number-char-cols=20 --auto-generate-sql --verbose –ssl-mode=DISABLED

pada HTTPS, Perintah yang digunakan sama dengan penggunaan pada HTTP namun disini opion –ssl-mode dihapus agar ssl otomatis diaktifkan kembali. Maka hasil yang didapatkan sebagai berikut.

Sehingga penggunaan SSL bisa mempengaruhi tingkat performa dari proses yang dilakukan, namun sebanding dengan tingkat keamanan yang telah diterapkan.
